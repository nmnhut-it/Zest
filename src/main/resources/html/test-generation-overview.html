ZEST Test Generation Guide

Multi-Agent System for Automated Test Creation

How It Works

ZEST uses a multi-agent system with a 7-phase process. Each agent handles a specific task in test generation, from analyzing context to fixing compilation errors.

Comparison with Traditional Approaches

Feature Manual/ChatGPT ZEST Approach
Project Understanding No context about codebase Analyzes project structure, dependencies, and class relationships
Workflow Single response, no iteration 7-phase process with state management
Error Fixing Manual correction required Automated compilation error detection and fixing

7-Phase Process

Each phase has a specific purpose in the test generation workflow:

Phase 1: INITIALIZING

Sets up the environment, initializes agents, and establishes LLM connection. Creates a unique session for tracking.

Phase 2: GATHERING_CONTEXT

Context Agent analyzes:

  • Class structure and methods to test
  • Dependencies and external services
  • Existing test patterns in the project
  • Build configuration
File Analysis Dependency Mapping
Phase 3: PLANNING_TESTS

Coordinator Agent designs test strategy:

  • Identifies test types needed (unit, integration, edge cases)
  • Creates test scenarios with descriptions
  • Selects appropriate testing tools
Smart Planning
Phase 4: USER_SELECTION

Displays proposed test scenarios. Users can:

  • Select which scenarios to implement
  • Skip unnecessary tests
  • Add custom requirements
Phase 5: GENERATING_TESTS

Test Writer Agent creates test code:

  • Follows project conventions
  • Uses descriptive test names
  • Sets up test data and mocks
  • Adds meaningful assertions
@Test void calculateTotal_whenValidItems_returnsCorrectSum() { // Given List<Item> items = Arrays.asList( new Item("A", 10.00), new Item("B", 20.00) ); // When double total = calculator.calculateTotal(items); // Then assertEquals(30.00, total); }
Phase 6: MERGING_TESTS

Merger Agent combines test methods:

  • Removes duplicate code
  • Optimizes setup/teardown
  • Organizes test structure
Phase 7: FIXING_TESTS

Fixing Agent resolves compilation issues:

  • Adds missing imports
  • Fixes syntax errors
  • Adjusts method signatures

Testing Strategies

Unit Testing

Creates isolated tests for individual methods using mocks for dependencies.

Integration Testing

When appropriate, uses tools like Testcontainers for database testing:

@Container static PostgreSQLContainer postgres = new PostgreSQLContainer<>("postgres:15") .withDatabaseName("testdb");

API Testing

Mocks external APIs using WireMock when needed:

@RegisterExtension static WireMockExtension wireMock = WireMockExtension.newInstance() .options(wireMockConfig().port(8089)) .build();

FAQ

Can I customize the test style?

ZEST analyzes existing tests in your project and follows similar conventions.

Does it support Kotlin?

Yes, both Java and Kotlin are supported.

How much does it cost in LLM tokens?

Token usage is phase-based, allowing you to control costs by selecting which tests to generate.

What languages are supported?

Currently Java and Kotlin, with more languages planned.

Summary

ZEST Test Generation uses a multi-agent architecture with a 7-phase process, integrated directly into your IDE. The system analyzes context, suggests test scenarios, and automatically fixes compilation errors.
