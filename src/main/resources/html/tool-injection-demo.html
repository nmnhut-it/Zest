Zest Tool Settings Injection

Automatically inject Zest tools into OpenWebUI settings

✨ Updated Approach: Tools are now injected using the OpenWebUI settings update API. This approach doesn't interfere with other features like web search and provides persistent tool configuration.

How It Works

The tool injection system uses the OpenWebUI settings update API to properly persist Zest tool servers to your configuration. This happens when:

  • You enable tool injection (tools are immediately added)
  • The settings are properly persisted across sessions
  • Tools are available in all conversations
  • No interference with other OpenWebUI features

Expected Settings Structure

When enabled, the following tool server will be added to your settings:

{ "url": "http://localhost:8765/zest", "path": "openapi.json", "auth_type": "bearer", "key": "", "config": { "enable": false, "access_control": {} }, "info": { "name": "Zest Code Explorer", "description": "Code exploration tools from IDE" } }

Testing Instructions

  1. Click "Enable Tool Injection" above
  2. Navigate to OpenWebUI Settings → Connections
  3. Look for "Zest Code Explorer" in the Tool Servers section
  4. The tool should appear automatically without manual configuration
  5. Use the tool in any OpenWebUI conversation
